     PRODUCT REQUIREMENTS DOCUMENT (PRD) 
   Nombre del Proyecto 
Sistema Web de Evaluación y Analítica de Satisfacción para Establecimientos de 
Consumo Universitario 
Universidad: Universidad Anáhuac Oaxaca 
Materia: Software II 
Duración: 8 semanas 
Equipo: 3 integrantes 
 
   Visión del Producto 
Desarrollar un sistema web transaccional con módulo analítico integrado que permita: 
• Capturar reseñas estructuradas de estudiantes 
• Analizar la satisfacción mediante métricas cuantitativas 
• Aplicar análisis de sentimiento con Machine Learning 
• Transformar datos operativos en indicadores estratégicos para gerentes 
No es una app tipo Yelp. 
Es un sistema de inteligencia operativa universitaria. 
 
   Problema que Resuelve 
Actualmente los establecimientos universitarios: 
• No cuentan con métricas estructuradas de satisfacción 
• No poseen análisis temporal de desempeño 
• No tienen indicadores estratégicos derivados de datos 
• No pueden detectar patrones de insatisfacción automáticamente 
El sistema centraliza, estructura y analiza información para apoyar la toma de decisiones.

     Módulo Analítico 
ETL 
Extract: 
• Reviews desde PostgreSQL 
Transform: 
• Limpieza de texto 
• Normalización 
• Tokenización 
Load: 
• Guardado en tablas analíticas 
 
Modelo de Sentimiento 
Pipeline: 
Texto → TF-IDF → Logistic Regression 
Split 80/20 
Métricas: 
• Accuracy 
• Precision 
• Recall 
• F1 
 
     Índice Global de Experiencia (IGE) 
Fórmula: 
IGE = 100 * ( 
0.5 * food_norm + 
0.3 * service_norm + 
0.2 * price_norm 
) 
Justificación: 
• Comida = variable primaria de satisfacción 
• Atención = impacto en retención 
• Precio = impacto contextual 
 
     Orquestación Analítica 
node-cron ejecuta job programado 
Backend ejecuta pipeline Python 
Python guarda métricas en DB 
Node registra versión del modelo
